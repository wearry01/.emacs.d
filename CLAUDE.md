# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## About

This is a personal Emacs configuration, managed as a git repo rooted at `~/.emacs.d`. It targets macOS and is written in Emacs Lisp with `lexical-binding: t`.

There is no build step, linter, or test suite. The "run" step is loading Emacs; errors appear in the `*Messages*` buffer.

## Architecture

`init.el` adds `lisp/` to `load-path` and `require`s each module. Modules are self-contained and use `use-package` with `:ensure t` for package installation.

| File | Purpose |
|---|---|
| `lisp/init-packages.el` | Package archives (TUNA mirrors), `use-package` bootstrap, auto-package-update (45-day interval), emacs server |
| `lisp/init-appearance.el` | Monaco font, `timu-macos-theme`, doom-modeline, disable chrome |
| `lisp/init-general.el` | macOS modifier keys (opt=meta, cmd=super), completions (vertico+orderless+marginalia+company), consult, which-key, smartparens |
| `lisp/init-evil.el` | Evil modal editing, evil-collection, evil-nerd-commenter. Leader key `SPC` for window/buffer commands. Consult integrated for search (`/`, `'`, `C-b`, `C-/`, `C-<return>`) |
| `lisp/init-org.el` | Org mode: agenda (beorg/iCloud path), capture, habit tracking, babel (dot, julia, emacs-lisp, shell), LaTeX preview via dvisvgm at 1.8x scale, ctex LaTeX class, ox-beamer, org-fragtog |
| `lisp/init-org-ext.el` | org-roam (Zettelkasten in `~/Documents/Zettelkasten/roam-notes/`), citar bibliography (`~/Documents/Zettelkasten/ref-lib.bib`), ox-hugo blog export, weekly journals, log archiving |
| `lisp/init-lsp.el` | LSP support via lsp-mode + lsp-ui with flycheck. Active in LaTeX, C/C++, and Python modes. Prefix `s-l` |
| `lisp/init-llm.el` | ellama with ollama backend (phi4-mini, qwen3.5:4b, nomic-embed-text). Prefix `C-c e`. **Not loaded in init.el** — requires manual `require` to enable |
| `lisp/init-tex.el` | AUCTeX with RefTeX, cdlatex, preview-dvisvgm, Skim PDF viewer via `displayline`, XeLaTeX/LuaLaTeX commands |
| `lisp/custom.el` | Auto-generated by Emacs Customize. Machine-specific; do not hand-edit |

`lisp/init-meow.el` exists but is **not loaded** — evil is the active modal editor.

## Key conventions

- New packages are added via `use-package` in the relevant module, never in `init.el` directly
- `custom-set-variables` / `custom-set-faces` go in `lisp/custom.el` (Emacs Customize writes there automatically)
- macOS-specific paths: Skim (`/Applications/Skim.app`), iCloud (`~/Library/Mobile Documents/…`), mdfind used for `consult-locate`
- Personal variables use the `wearry/` namespace prefix
- Org agenda files live in iCloud for beorg mobile sync
